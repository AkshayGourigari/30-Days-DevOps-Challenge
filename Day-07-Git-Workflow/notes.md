# 🚀 Day 7 - Git & GitHub Deep Dive

---

## 📌 Introduction

Git is a distributed version control system used to track changes in code.

GitHub is a platform used to host and manage Git repositories.

---

## 📌 Git Architecture

Working Directory → Staging Area → Local Repository → Remote Repository

- Working Directory: Where files are created and modified
- Staging Area: Prepares files for commit
- Local Repository: Stores commits locally
- Remote Repository: Stores code on GitHub

---

## 📌 Core Concepts

### 🔹 Version Control
Tracks changes and allows rollback

### 🔹 Repository
A place where project files and history are stored

### 🔹 Commit
A snapshot of changes saved in Git

---

## 📌 Branching

A branch is a separate working environment.

### Why use branches?
- Avoid breaking main code
- Work independently
- Support team collaboration

---

## 📌 Merging

Merging combines changes from one branch into another.

---

## 📌 Best Practices

- Never work directly on main branch
- Always use feature branches
- Write meaningful commit messages
- Pull latest code before working

---

## 📌 Common Mistakes

- Skipping git add
- Working on main branch
- Not pulling latest changes
- Large unclear commits

---

## 📌 My Practice

- Created branch
- Committed changes
- Pushed to GitHub
- Merged into main

---

## 📌 Key Takeaway

Git is not just commands — it is a workflow used in real DevOps projects.
